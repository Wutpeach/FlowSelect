## 亮点
- 这是 `0.3.0` 正式发布前的第五个测试打包版本，重点验证一组 macOS 交互与视频下载体验修复：主窗口双击打开目标文件夹、右键菜单定位，以及 YouTube / Bilibili 注入下载链路的稳定性。

## 修复
- 修复了 macOS 主窗口双击打开目标文件夹不生效的问题，并补上更稳定的双击触发兜底，避免面板事件只在部分环境下命中。
- 修复了 macOS 右键菜单弹出位置偏移的问题，菜单现在应当出现在鼠标点击附近，不再跑到主窗口外。
- 修复了 YouTube / Bilibili 注入下载链路里临时 cookies 文件错误写到只读目录的问题，macOS 打包版不应再因 `EROFS: read-only file system` 而在准备阶段直接失败。
- 优化了 yt-dlp 在解析媒体阶段的状态反馈，YouTube / Bilibili 在真正开始下载前不再长时间只显示 `Preparing...`，而会更早进入可感知的解析/下载中状态。

## 说明
- 这是预发布测试标签，主要用于复测 macOS 上三类高频问题：主窗口双击交互、右键菜单定位，以及通过注入播放器按钮发起的视频下载。
- 本次 GitHub Release 预期包含 `FlowSelect_0.3.0-rc5_browser_extension.zip`，用于继续验证桌面端与扩展联动链路在预发布版本中的一致性。
- macOS 本次重点验证 arm64 设备：双击主窗口能打开目标文件夹、右键菜单位置正确、YouTube / Bilibili 注入按钮下载不再因 cookies 临时文件路径问题失败，且下载前的等待阶段文案更符合实际状态。
