## Highlights
- Added a clearer runtime dependency flow so FlowSelect can surface downloader/runtime setup status in the main window and Settings, with better guidance when required components still need attention.
- Improved packaged runtime delivery by advancing the managed bootstrap flow for bundled tools such as the Pinterest sidecar and Deno, reducing machine-specific setup drift across packaged installs.
- macOS releases now ship as unsigned open-source DMGs with a standard drag-to-`Applications` layout, a bundled install guide, and a `Fix Open.command` helper for first-launch recovery.

## Fixes
- Improved `Highest` download retry preflight and success-state handling so downloads are less likely to get stuck in misleading states after runtime or network interruptions.
- Hid low-level cancel-process commands from the normal app surface and hardened macOS yt-dlp/runtime checks to make packaged behavior more predictable.

## Notes
- The macOS DMGs in this release are intentionally unsigned open-source builds and do not require Apple Developer Program signing. Users should drag `FlowSelect.app` into `Applications` first, then use Finder `Open`, `Privacy & Security > Open Anyway`, or the bundled `Fix Open.command` helper if Gatekeeper blocks first launch.
- This release also adds the runtime-sidecar publish pipeline used to keep bundled downloader assets aligned with application expectations.
