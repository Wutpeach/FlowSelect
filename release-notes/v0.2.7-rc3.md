## Highlights
- Simplified desktop release assets so Windows now centers on a single `setup.exe` installer, while user download packages and updater-only archives use clearer names.
- Aligned local builds and GitHub Actions to the same platform-aware Tauri packaging defaults so Windows release candidates no longer generate or publish `MSI` bundles by default.

## Fixes
- Removed the extra Windows `MSI` publishing path that made the release page look like it had multiple equivalent installers for the same platform.
- Renamed macOS updater archives and user-facing DMG / portable artifacts so updater internals are easier to distinguish from packages meant for manual download.

## Notes
- This is a prerelease build intended to validate the updated packaging and GitHub Actions release flow before promoting the asset naming changes more broadly.
- Windows in-app updater still targets the signed `setup.exe` installer path; the change here is packaging clarity, not a switch in updater transport.
