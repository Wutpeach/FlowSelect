## Highlights
- Restored the Windows right-click menu after the inline overlay regression prevented it from opening reliably.
- Fixed the `Set Output Folder` action in the Windows context menu so it can open the native folder picker again.

## Fixes
- Context-menu folder actions now close the menu before launching native Windows behavior.
- The folder picker path is handled through the Tauri backend, which avoids the cross-window focus and z-order issues that were blocking this flow.
- Added an explicit contract for context-menu folder actions so future changes keep the same close, focus, and persistence behavior.

## Notes
- This release also includes recent UI polish and extension workflow improvements that landed after `v0.2.0`.
- The main focus of `0.2.2` is stability on Windows context-menu interactions.
