## Highlights
- Added an in-app FlowSelect update entry in the main window so packaged builds can move toward application self-update instead of only surfacing the standalone `yt-dlp` updater.
- Wired the desktop build and release pipeline for Tauri updater artifacts, including signed installer metadata and the public `latest.json` manifest expected by updater clients.

## Fixes
- Kept the `Settings` page downloader section focused on `yt-dlp` / `pin-dlp` / runtime management so the new app-update flow does not regress the existing downloader maintenance UI.
- Fixed the GitHub Release flow for immutable repositories by creating prereleases as drafts first, uploading assets, and only then publishing the release.

## Notes
- This is a prerelease build intended to validate the GitHub Actions release pipeline and updater artifacts, not a general end-user upgrade target.
- Existing stable `v0.2.7` clients will not auto-discover `v0.2.7-rc2` through the current `/releases/latest/download/latest.json` endpoint because GitHub `latest` ignores prereleases and `0.2.7` is semver-higher than `0.2.7-rc2`.
- After the workflow succeeds, verify that the GitHub Release contains the public `latest.json` updater manifest and the expected installer/archive assets before promoting the updater flow further.
