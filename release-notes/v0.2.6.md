## Highlights
- Added a two-stage queue in the desktop main window that separates downloads from transcoding, shows task-type indicators in the queue badge, and lets failed transcodes be retried or removed directly.
- Simplified the browser extension popup by removing the standalone `AE Format` toggle and keeping `Highest / Balanced / Saver` as the only download choices, with a contextual hint when `Highest` may trigger automatic transcoding.
- Added a main-window double-click shortcut to open the current output folder while keeping the existing context-menu folder actions available.

## Fixes
- Restored download quality selection so `Highest`, `Balanced`, and `Saver` map more reliably to the intended yt-dlp runtime behavior.
- Fixed `Highest` YouTube downloads that could fall back to lower-tier HLS selections by improving selected-format probing and retry behavior around cookie-backed sessions.
- Fixed AE-safe media probing to use the app-managed `ffprobe` runtime, which prevents already compatible downloads from entering the transcode queue just because the host machine lacks a global `ffprobe`.

## Notes
- This release focuses on making the new download-to-transcode workflow easier to understand while stabilizing the first regression wave after the queue redesign.
